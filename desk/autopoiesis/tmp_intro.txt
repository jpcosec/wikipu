La idea central: "Compilar" en lugar de "Buscar"
Karpathy critica que el RAG estándar es "sin memoria" o "sin acumulación". Cada vez que haces una pregunta, el sistema busca trozos de texto sueltos y trata de unirlos. Su propuesta es:

Interfaz: El usuario no interactúa con los archivos originales, sino que navega por esta Wiki (usando herramientas como Obsidian) o le pregunta al LLM, que ahora tiene una estructura de conocimiento coherente para consultar.

Karpathy utilizó una analogía muy clara en su Gist y publicaciones de seguimiento:

Obsidian es el IDE (el entorno de desarrollo).

El LLM es el programador (quien mantiene y actualiza los archivos).

La Wiki es el código fuente (el conocimiento estructurado).

¿Por qué es importante?
Según Karpathy, para proyectos personales o de departamentos (~100 artículos), no necesitas bases de datos vectoriales complejas. Un LLM es lo suficientemente inteligente como para navegar por una estructura de carpetas y archivos Markdown bien organizados. Además, esto permite que el conocimiento sea propiedad del usuario (archivos locales) y no dependa de un proveedor de nube específico.
+2

from pydantic import BaseModel, Field
from typing import Literal

class ComplianceLevel(BaseModel):
    status: Literal["planned", "designed", "scaffolding", "mocked", "implemented", "tested"]
    failing_checks: list[str] = Field(description="List of documentation or test rules not met")

class ASTNodeSignature(BaseModel):
    node_type: Literal["pydantic_model", "graph_node", "determinisic_tool", "abc"]
    dependencies: list[str] = Field(description="List of internal contract names this node depends on")

class KnowledgeNode(BaseModel):
    """
    Representa una entidad en la Wiki. Puede ser un concepto de negocio, 
    un componente técnico o un módulo completo.
    """
    entity_id: str = Field(description="Unique identifier (e.g., 'core.ai.generate_documents')")
    description_from_code: str = Field(description="Extracted directly from the module docstring")
    ast_signature: ASTNodeSignature | None
    compliance: ComplianceLevel
    linked_future_docs: list[str] = Field(description="List of future_docs files affecting this node")
