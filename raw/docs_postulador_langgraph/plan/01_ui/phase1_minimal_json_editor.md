# UI Phase 1: Minimal JSON Editor (NodeEditor)

## Objective
Connect the current `NodeEditor` directly to the local JSON files generated by the pipeline. Delay Neo4j adoption until the LLM pipeline is 100% stable and job volume demands it.

## Key Milestones
1. **Knowledge-Tree Review:** Load/Save `extract_understand/approved/state.json`.
2. **Match Review:** Visualize and adjust `Requirement` -> `Match` -> `Evidence` scores.
3. **Redaction Review:** Edit document "deltas" and approve final Markdown/JSON outputs.

## Architecture Rules
- **No Neo4j:** UI communicates with a lightweight FastAPI backend for direct File I/O on the `data/` folder.
- **Representation Schema:** Agnostic frontend mapping for node visualization.
