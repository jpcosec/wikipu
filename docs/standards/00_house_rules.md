# 🏛️ 00_house_rules — Ecosystem Base Rules

This document defines the immutable laws of the repository. Every agent, LLM, or human developer operating on this code must comply with these regulations to ensure our Knowledge Graph (The Wiki) remains 100% accurate.

> **Supreme Principle:** Code is the Truth. Documentation is the Why. The Wiki is the Index.

---

## ⚖️ 1. Law of Separation of State and Semantics
It is strictly forbidden to mix transient data (execution) with semantic data (knowledge). No logs or IDs are to be written in Markdown.

## ⚖️ 2. Law of the Single Source of Architectural Truth
Static diagrams in READMEs are prohibited. The only official representation of the architecture is the Meta-Graph generated by the `wiki_compiler`.

## ⚖️ 3. Law of Code Visibility
All I/O contracts must be `Pydantic` models. Using plain dictionaries (`dict`) to pass messages between modules is forbidden.

## ⚖️ 4. Law of Structural Integrity (Scaffolding CLI)
No human or agent shall create a module skeleton by hand. Every new module must be initialized using the CLI: `wiki-compiler scaffold --module "path"`. This ensures mandatory `contracts.py`, `__init__.py`, and `README.md`.

## ⚖️ 5. Law of Historical Memory (ADRs)
Every major design decision or discarded technology must be recorded in `docs/adrs/`.

## ⚖️ 6. Law of Declared Flexibility (.wikiignore)
If legacy scripts cannot comply with these House Rules, declare technical debt using a `.wikiignore` file or the `@wiki_exempt` decorator.

## ⚖️ 7. Law of the Ubiquitous Glossary
Every central concept must reside in `docs/domain_glossary.yaml` to avoid polysemy (e.g., "User" vs. "Customer").

## ⚖️ 8. The CI/CD Guardian
The `wiki_compiler` runs on every PR. If the *Compliance Score* decreases, the merge is blocked.

## ⚖️ 9. Law of Orthogonality (Closed Loop)
No one can create a new module without submitting a `TopologyProposal` to the compiler. For agents, there is a strict limit of **3 attempts** to achieve orthogonality. If they fail, human intervention is required.

## ⚖️ 10. Law of the Sanctuary (The Seed/Raw Directory)
The `raw/` or `seed/` directory is the immutable seed. It is **read-only** for agents and contains the fundamental requirements or source documents that bootstrap the graph.

## ⚖️ 11. Law of Atomicity and Transclusion (DRY Wiki)
Explaining a concept twice is forbidden. Shared concepts are extracted to an atomic node (`src/wiki/concepts/`) and embedded using transclusion (`![[concept_name]]`).

## ⚖️ 12. Law of Static Compilation (Baking)
Transclusion syntax (`![[...]]`) is only valid in the Wiki source (`src/wiki/`). The CI must execute `wiki-compiler build` to render standard, readable Markdown in the `docs/` folder and update the Graph database for Agents.
