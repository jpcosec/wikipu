## Details

Code-derived nodes (file:, code:, dir:) are regenerated by wiki-compiler build.
If a function is deleted, its node disappears on the next build automatically.
That handles existence.

But the build does not detect architectural drift:
a module that duplicates another, a test that no longer covers the right surface,
a config file nothing reads. These are structural violations that persist
across builds because the code compiles fine — they are not syntax errors,
they are knowledge errors.

Cleansing addresses the knowledge layer, not the compilation layer.
